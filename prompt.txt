Нужно написать приложение-обёртку для инструмента обхода DPI. Сам инструмент берётся из гитхаб репозитория, пример последней версии такого инструмента лежит в папке zapret. Название - ZPRT App.
Стек - tauri + react.

Структура проекта должа быть читаемой для разработчика, чтобы я мог в сулчае необходимости сам внести какие-то правки/исправить какой-то дефект.

Суть инструмента, управление которым необходимо вынести в отдельное приложение:
- В основном инструмент состоит из набора bat-файлов (стратегий), запуск которых приводит обработке трафика устройства таким образом, чтобы обойти DPI.
- Файлы с паттерном наименования general*.bat используются непосредственно для запуска программы обхода. При запуске такого bat-файла проиходит открытие winws.exe из папки bin с определёнными параметрами. Далее окно командной строки изначального bat файла закрывается и открывается окно CMD уже самого winws.exe. Пока окно winws открыто - обход работает.
- В папке lists расположены тектовые документы для хранения списков ip/доменов, для которых winws будет осуществлять модификацию трафика в зависимости от запущенной стратегии.
- Также существует файл service.bat для включения автозапуска и настроек других функций, далее их список и описание:

Install Service - установка любой стратегии в автозапуск (services.msc)
Remove Services - удаление стратегии и WinDivert из служб
Check Status - проверка статуса обхода и служб (стратегии на автозапуске и WinDivert)
Game Filter - переключение режима обхода для игр (и других сервисов, использующих UDP и TCP на портах выше 1023).
После переключения требуется перезапуск стратегии.
В скобках указан текущий статус (включено/выключено).
IPSet Filter - переключение режима обхода сервисов из ipset-all.txt.
Полезно при тестировании, если не работает ресурс, который без zapret работает
В скобках указан текущий статус:
none - никакие айпи не попадают под проверку
loaded - айпи проверяется на вхождение в список
any - любой айпи попадает под фильтр
Auto-Update Check - Вкл/Выкл автоматическую проверку на обновления
Update IPSet List - обновление списка ipset-all.txt актуальным из репозитория
Update Hosts File - обновление файла hosts для починки веб версии телеграма и подключения к голосовому чату Discord
Check for Updates - проверка на обновления
Run Diagnostics - диагностика на распространённые причины, по которым zapret может не работать.
В конце можно очистить кэш  Discord, что может помочь, если он неожиданно перестал работать
Run Tests - запуск утилиты для проверки стратегий на работоспособность:
Standard tests - проверка сайтов из utils/targets.txt
DPI checkers - проверка DPI на различных провайдерах (Cloudflare, Amazon и др.)

Для глубокого понимания работы инструмента сам изучи файлы инструмента в папке zapret.

Нужно написать приложение, которое позволит не взаимодействовать со всеми этими файлами напрямую. Но подход должен быть такой, что приложение как бы будет являться обёрткой поверх инструмента обхода - без внесения правок в файлы проекта. Исключением будет являться только папка lists.
Для хранения каких-то кастомных данных/настроек пользователя можно использовать какой-нибудь конфиг-файл рядом с exe приложения ZPRT App.

Для начала нужно реализовать следующий функционал:
Программа при запуске должна отображать свой основной интерфейс. В нём должны быть следующие функции:
1. При отсутствии скачать и распаковать в папку zapret/версия_утилиты последнюю версию инструмента из github - https://github.com/Flowseal/zapret-discord-youtube/releases/latest. Нужный нам Asset - zapret-discord-youtube-*.zip, где вместо * обычно указана версия.
2. При наличии какой-либо версии запрета необходимо проверить актуальность версии с возможностью обновить его до последней. Проверка актуальной версии также происходит через запросы в гитхаб репозитория.
3. Возможность переключаться между установленными версиями утилиты запрета.
4. Возможность выбора стратегии, которая будет использоваться. Список доступных стратегий должен определяться динамически, в зависимости от набора файлов general*.bat в выбранной версии запрета.
5. Функция запуска/остановки обхода. Запускать нужно не через автозапуск, а с помощью bat-файлов стратегий. Также после запуска нужно визуально скрывать окно командной строки, которое появляется при запуске generat-файлов. Стоит учесть, что это не то же окно cmd, которое выводится при запуске bat, а новое, которое пренадлежит непосредственно winws.exe. 
6. Функция включения/отключения автозапуска обхода при включении ПК. Работать должно не через Install Service из service.bat, а посредством автоматического запуска самого приложения ZPRT App.
7. Возможность редактирования списков доменов для обхода внтури приложения - lists/list-general.txt и lists/list-exclude.txt.
8. Вывод иконки приложения в трей меню windows с возможностью управления основными функциями ZPRT App через ПКМ-меню. При закрытии приложения на крестик оно не должно закрываться полностью, иконка в трей-меню должна оставаться. При ЛКМ по иконке в трее должно открываться основное окно приложения.

Извучи мои инструкции, изучи структуру инструмента обхода в папке zapret. При необходимости задай мне необходимые уточняющие вопросы.
